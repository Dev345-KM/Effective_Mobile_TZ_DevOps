# Effective_Mobile_TZ_DevOps
------------------------------
# Как запустить проект
Для запуска проекта убедитесь, что у вас установлены Docker и Docker Compose.

Клонируйте репозиторий:
git clone https://github.com/Dev345-KM/Effective_Mobile_TZ_DevOps.git
cd Effective_Mobile_TZ_DevOps

Запустите сборку и контейнеры:
docker-compose up -d --build

Проверьте статус контейнеров:
docker-compose ps
----------------------------------
# Как проверить результат
После того как все контейнеры перейдут в статус Up, выполните следующие действия:

В браузере: Перейдите по адресу http://localhost. Вы должны увидеть ответ от вашего приложения или приветственную страницу.

Через терминал (curl):
curl -I http://localhost

Проверка логов: Чтобы убедиться, что запросы доходят до бэкенда через Nginx:
docker-compose logs -f
------------------------------------------------------
# Как работает схема (Nginx -> Backend)
Архитектура проекта построена по принципу Reverse Proxy:

Nginx (EntryPoint): Выступает в роли единой точки входа. Он принимает входящий трафик на порту 80 и перенаправляет его во внутреннюю сеть Docker.

Проксирование: Внутри конфигурации Nginx прописана директива proxy_pass, которая указывает на имя сервиса backend. Docker DNS автоматически разрешает это имя в IP-адрес соответствующего контейнера.

Backend: Принимает запрос от Nginx, обрабатывает его и отправляет результат обратно.

Безопасность и масштабируемость: Бэкенд скрыт от прямого доступа извне, что позволяет легко менять количество воркеров без изменения внешних настроек.
